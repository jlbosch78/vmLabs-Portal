# Política de seguridad (SECURITY)

Gracias por ayudar a mantener este proyecto seguro.

## Versiones soportadas
Este repositorio no mantiene una matriz formal de versiones. Se recomienda usar la última versión publicada en `main` (o el último tag/release).

## Reporte de vulnerabilidades
Por favor **no** abras incidencias públicas (Issues) para reportar vulnerabilidades.

En su lugar, utiliza uno de estos métodos:

1) **GitHub Security Advisories** (recomendado)  
   - Ve a la pestaña **Security** del repositorio → **Advisories** → **New draft security advisory**.

2) **Correo privado**  
   - Envía un email al mantenedor del repositorio (indica en el README o en el perfil de GitHub un correo de contacto).
   - Incluye en el asunto: `Security report: VM Labs Portal`.

## Qué incluir en el reporte
- Descripción del problema e impacto
- Pasos para reproducir (PoC mínimo si aplica)
- Versión/commit afectado (hash, tag o fecha)
- Logs relevantes (sin secretos)
- Sugerencia de mitigación (si la tienes)

## Tiempos orientativos de respuesta
- Confirmación de recepción: 3–7 días
- Evaluación inicial: 7–14 días
- Publicación de fix: dependerá de la complejidad

## Buenas prácticas
- Nunca incluyas contraseñas, tokens o el contenido del `.env` en el reporte.
- Si el problema afecta a credenciales expuestas, rota inmediatamente.
